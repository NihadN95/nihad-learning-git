
# Git

Git is a distributed version control system that helps you track changes in your code over time.

## What Git Does

- **Tracks Changes**: Records modifications to files, allowing you to see what changed and when
- **Enables Collaboration**: Multiple developers can work on the same project simultaneously
- **Maintains History**: Keeps a complete history of your project, so you can revert to previous versions
- **Branching**: Create isolated branches to work on features without affecting the main code
- **Merging**: Combine changes from different branches back into the main project
- **Distributed**: Each developer has a complete copy of the project history

## Basic Workflow

1. Make changes to your files
2. Stage changes with `git add`
3. Commit changes with `git commit`
4. Push to a remote repository with `git push`
5. Pull updates from others with `git pull`

## Getting Started

```bash
git init              # Initialize a new repository
git add .             # Stage all changes
git commit -m "message"  # Commit with a message
git push              # Push to remote
```
